if you don't look, you build for a system that doesn't map to reality — which is how most AI pilots fail.

phase 1 · discovery

sit with the work,
not the org chart.

a

shadow full days

in an hour people describe the job they think they have. beside them for eight, you see the exception actually happen.

b

get system access

ERP, CRM, the four spreadsheets, the Slack approvals. read the real artifacts, not the SOP.

c

expect multiple owners

several people run the "same" process differently. one source is a sample size of one.

d

chase the exceptions

one way to go right, a thousand ways to go wrong. the unwritten rules are the actual spec.

discovery is also where trust is built — the relationship is infrastructure for everything that follows.

phase 2 · audit

decide what should be automated —
and what should not.

deterministic software

when the rules and inputs are predictable. most of a good solution is this — don't spend a model on an if-statement.

·
an agent

when the objective is clear but the inputs, path, or actions vary. often only 3 steps of 10 need judgment.

·
a human in control

when the decision carries material ambiguity, accountability, or irreversible consequences.

some workflows shouldn't be touched at all. prioritize lengthy, high-volume workflows where the improvement is large enough to matter.

phase 2 · audit

a business measures
exactly three things.

1

revenue uplift

reach customers faster, drive more value per customer, unlock throughput the old process capped.

2

risk mitigation

fewer errors, auditable decisions, escalation by rule — the failures that were silent now leave evidence.

3

cost savings

hours returned, rework eliminated, cost per run measured and driven down.

every expected-value claim in the operating map lands in one of these buckets — quantified, not vibes.

phase 3 · build

build for the
unhappy paths.

there is one way something goes right and a thousand ways it goes wrong. an agent built for the happy path is worth nothing; the one that handles the exceptions is the one worth deploying.

build11 / 18
12
phase 3 · build

add capability in this order.

1

agent loop

prompt → model → response → next step, until done or a step limit hits.

2

tool use

start minimal — one API call, one search. the agent decides when to call.

3

guardrails

input validation · max-step limit · output filtering.

4

audit trail

log every prompt, tool call, result, error, timestamp. no traces, no trust.

5

structured outputs

a defined JSON schema, validated on every response. retry or escalate.

6

state & recovery

checkpoint every few steps. kill it — it must resume where it stopped.

7

failure handling

missing data, dead APIs, timeouts, duplicates — define behavior for each.

8

retries & backoff

exponential on every external call: 1s · 2s · 4s · 8s, capped. then escalate.

pick one model and one agent platform and get genuinely good at it — your value is judgment about the workflow, not the inference layer.

15
phase 5 · deployment

make it work inside the business.

1

integrate with what exists

never open with "first, migrate off your ERP." build on top of the current stack — that's the edge over rip-and-replace.

2

test in a controlled environment

a sandbox inside the company's own infrastructure, where you can run, inspect, and debug safely.

3

increase autonomy gradually

shadow → approve every action → approve only risky actions → autonomy where earned. smallest useful action first.

the pattern to push for: agent handles intake, validation, and drafting — a human approves — the system completes the rest.

phase 5 · deployment

deployment is a people problem
wearing a software costume.

you are a risk to your sponsor

status quo is safe; bringing you in is not — if it fails, it's their name on it. de-risk them: visible traces, human approval gates, gradual autonomy, measurable checkpoints. prove value before asking for trust.

your job is to get them promoted

drive value cost-effectively in the three buckets and make it attributable, so at review time they can point at it. face to face makes every hard conversation easier — the relationship is infrastructure.

when production breaks, it's your name on the line: know who to call, what to do, and how to fix it.
