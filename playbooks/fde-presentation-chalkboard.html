phase 1 · discovery

Sit with the work, not the org chart.

Shadow full days

In an hour, people describe the job they think they have. Beside them for eight, the real exception happens.

Get system access

ERP, CRM, the four spreadsheets, the Slack approvals. Read the real artifacts, not the SOP.

Expect many owners

Several people run the "same" process differently. One source is a sample of one — sit with more.

Chase exceptions

One way to go right, a thousand to go wrong. The unwritten rules are the actual spec.

Discovery is where trust is built — the relationship is infrastructure for everything that follows.

Board 09 / 18
phase 2 · audit

Decide what should be automated — and what should not.

Deterministic software

When rules and inputs are predictable. Most of a good solution is this — don't spend a model on an if-statement.

An agent

When the objective is clear but inputs, path, or actions vary. Often only 3 of 10 steps need judgment.

A human in control

When the decision carries material ambiguity, accountability, or irreversible consequences.

Some workflows shouldn't be touched at all. Prioritize lengthy, high-volume ones where the improvement is large enough to matter.

Board 10 / 18
phase 2 · audit

A business measures exactly three things.

1

Revenue uplift

Reach customers faster, drive more value per customer, unlock throughput the old process capped.

2

Risk mitigation

Fewer errors, auditable decisions, escalation by rule — silent failures now leave evidence.

3

Cost savings

Hours returned, rework eliminated, cost per run measured and driven down.

Every expected-value claim in the operating map lands in one of these buckets — quantified, not vibes.

Board 11 / 18
phase 3 · build

Build for the unhappy paths.

One way something goes right, a thousand ways it goes wrong. An agent built for the happy path is worth nothing — the one that handles the exceptions is worth deploying.

Board 12 / 18
phase 3 · build

Add capability in this order.

1

Agent loop

Prompt → model → response → next, until done or max-step.

2

Tool use

Start minimal — one call, one search. The agent decides when.

3

Guardrails

Input validation · max-step limit · output filtering.

4

Audit trail

Log every prompt, call, result, error. No traces, no trust.

5

Structured out

A JSON schema, validated each response. Retry or escalate.

6

State & recovery

Checkpoint often. Kill it — it must resume where it stopped.

7

Failure handling

Missing data, dead APIs, timeouts, dupes — behavior for each.

8

Retries + backoff

Exponential: 1·2·4·8s, capped. Escalate when they run out.

Pick one model and one agent platform and get genuinely good at it — your value is judgment about the workflow, not the inference layer.

Board 15 / 18
phase 5 · deployment

Make it work inside the business.

1

Integrate with what exists

Never open with "first, migrate off your ERP." Build on top of the current stack — that's the edge over rip-and-replace.

2

Test in a controlled env

A sandbox inside the company's own infrastructure, where you can run, inspect, and debug safely.

3

Increase autonomy gradually

Shadow → approve every action → approve only risky ones → autonomy where earned. Smallest useful action first.

The pattern to push for: agent handles intake, validation, drafting — a human approves — the system completes the rest.

Board 16 / 18
phase 5 · deployment

Deployment is a people problem wearing a software costume.

You are a risk to your sponsor

Status quo is safe; bringing you in is not — if it fails, it's their name on it. De-risk them: visible traces, approval gates, gradual autonomy, measurable checkpoints. Prove value before asking for trust.

Your job is to get them promoted

Drive value in the three buckets and make it attributable, so at review time they can point at it. Face to face makes every hard conversation easier — the relationship is infrastructure.

When production breaks, it's your name on the line: know who to call, what to do, how to fix it.
