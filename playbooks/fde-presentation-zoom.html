The premise

Everyone can buy intelligence.
That can't be the moat.

The same foundational model is available to anyone able to pay. So the edge is no longer who has intelligence — it's where, how, and why they use it. Someone has to decide where intelligence belongs, and where it does not.

Audit
Discovery · Map reality
Map the real process, not the documented one.

The documented process is rarely the real process. Sit with the work, not the org chart — shadow full days, get system access, chase the exceptions. Then ship one operating map: current-state → future-state, a selected use case, boundaries, and expected value. Decide what to automate.

Build
For the unhappy paths
One way it goes right. A thousand ways it goes wrong.

An agent built for the happy path is worth nothing; the one that handles exceptions is the one worth deploying. Add capability in order — and pick one model and one platform, then get genuinely good at it.

3
Evals
Non-determinism → evidence
A golden dataset of real cases, hand-labeled.

Grade every run four ways — right data, required steps, matches the expert, safe to act. Cover the normal case, the edge case, incomplete information, an ambiguous request, and a high-risk action that escalates by rule. No evidence, no autonomy — the suite is what earns the system its next level of authority.

Right dataRequired stepsMatches expertSafe to act
4
Deploy
Make it work inside
Build on top of the stack that exists — never rip-and-replace.

Integrate with what's there, test in a controlled sandbox, then increase autonomy gradually: shadow mode → approve every action → approve only the risky ones → autonomy where it's earned. The pattern to push for: the agent handles intake, validation, and drafting; a human approves; the system completes the rest.

Shadow modeApprove allApprove riskyEarned autonomy
5
Observe
Watch production
Metrics, failure categories, cost per run.

Production is the real eval set. Watch what the system actually does, categorize every failure honestly, and feed it straight back into the golden dataset and the harness. Investigating failures is the work — that's what turning non-determinism into evidence means.

MetricsFailure categoriesCost per run
6
Improve
↻ The loop runs again
Fixing one workflow exposes the next bottleneck.

Bottlenecks are interconnected: repair one workflow and the constraint upstream appears while capacity frees downstream. The next engagement is usually visible from inside the first — that's how one workflow becomes the whole business, and the loop returns to Audit.

Audit · the ROI matrix

A business measures three things.

1
Revenue uplift  reach customers faster, unlock capped throughput
2
Risk mitigation  fewer errors, auditable decisions, escalation by rule
3
Cost savings  hours returned, rework eliminated, cost per run driven down

Every expected-value claim lands in one of these — quantified, not vibes.

Deploy · adoption & trust

A people problem in a software costume.

You're a risk to your sponsor — the status quo is safe, bringing you in is not. De-risk them with visible traces, approval gates, and gradual autonomy. Your job is to get them promoted: make the value attributable, and remember the relationship is infrastructure.
